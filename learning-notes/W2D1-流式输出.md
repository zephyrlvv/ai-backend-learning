## 什么是流式输出？

### 前端视角理解

**普通请求（你现在的 CRUD）**：
```javascript
// 发送请求 → 等全部数据 → 一次性显示
const res = await axios.get('/todos')
// 等待...（可能几秒）
console.log(res.data)  // 一次性拿到全部数据
```

**流式请求（ChatGPT 效果）**：
```javascript
// 发送请求 → 边接收边显示
const eventSource = new EventSource('/stream')
eventSource.onmessage = (e) => {
  console.log(e.data)  // 收到一个字，显示一个字
}
```

**类比**：
- 普通请求 = 下载完整视频再看
- 流式请求 = 在线看视频，边缓冲边播放

## 为什么要用流式输出？

### AI 应用场景

**大模型生成文本**：
- 生成一篇文章需要 5-10 秒
- 用户等待体验差

**流式解决方案**：
- 生成第一个字就立即返回
- 用户看到"打字机"效果，感觉响应很快
- 提升用户体验

## Python 核心技术：yield（生成器）

### 一句话解释

`yield` = **暂停函数，返回一个值，下次从暂停处继续**

### 类比 JavaScript 生成器

```javascript
// JavaScript
function* numberGenerator() {
  yield 1
  yield 2
  yield 3
}

const gen = numberGenerator()
console.log(gen.next().value)  // 1
console.log(gen.next().value)  // 2
console.log(gen.next().value)  // 3
```

```python
# Python
def number_generator():
    yield 1
    yield 2
    yield 3

gen = number_generator()
print(next(gen))  # 1
print(next(gen))  # 2
print(next(gen))  # 3
```

### 流式输出的核心逻辑

```python
def stream_text():
    text = "你好，世界！"
    for char in text:
        yield char      # 生成一个字，暂停
        time.sleep(0.1) # 模拟生成延迟

# 使用
for char in stream_text():
    print(char, end='', flush=True)  # 你 好 ， 世 界 ！（逐字显示）
```

## FastAPI 实现流式输出

### 基础版本

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import time

app = FastAPI()

@app.get("/stream")
def stream():
    def generate():
        text = "这是一段流式输出的文字，会像打字机一样逐字显示。"
        for char in text:
            yield f"data: {char}\n\n"  # SSE 格式
            time.sleep(0.05)  # 每个字延迟 50ms
    
    return StreamingResponse(
        generate(),
        media_type="text/event-stream"  # SSE 协议
    )
```

### 关键概念

| 概念 | 解释 |
|------|------|
| `yield` | 生成一个字，暂停，等前端接收后再继续 |
| `StreamingResponse` | FastAPI 的流式响应包装器 |
| `text/event-stream` | SSE（Server-Sent Events）协议 |
| `data: xxx\n\n` | SSE 数据格式要求 |

## Vue3 接收流式数据

### 使用 EventSource（原生 API）

```typescript
const responseText = ref('')

const startStream = () => {
  const eventSource = new EventSource('http://localhost:8000/stream')
  
  eventSource.onmessage = (event) => {
    responseText.value += event.data  // 逐字追加
  }
  
  eventSource.onerror = () => {
    eventSource.close()  // 出错或结束时关闭
  }
}
```

### 模板显示


## 实战：模拟 AI 对话流式输出

### 后端（main.py 添加）

```python
import random

@app.get("/chat/stream")
def chat_stream(msg: str):
    """模拟 AI 流式回复"""
    
    def generate():
        # 模拟 AI 思考后生成回复
        responses = [
            f"收到你的消息：{msg}。这是一个很有意思的问题！",
            f"关于\"{msg}\"，我的看法是这样的...",
            f"你提到了{msg}，让我想想...",
        ]
        response = random.choice(responses)
        
        # 逐字返回
        for char in response:
            yield f"data: {char}\n\n"
            time.sleep(0.03)  # 每个字 30ms，模拟打字速度
        
        # 发送结束标记
        yield "data: [DONE]\n\n"
    
    return StreamingResponse(
        generate(),
        media_type="text/event-stream"
    )
```

### 前端

```typescript
const message = ref('')
const reply = ref('')
const isStreaming = ref(false)

const sendStreamMessage = () => {
  isStreaming.value = true
  reply.value = ''
  
  const eventSource = new EventSource(
    `http://localhost:8000/chat/stream?msg=${encodeURIComponent(message.value)}`
  )
  
  eventSource.onmessage = (e) => {
    if (e.data === '[DONE]') {
      eventSource.close()
      isStreaming.value = false
      return
    }
    reply.value += e.data
  }
  
  eventSource.onerror = () => {
    eventSource.close()
    isStreaming.value = false
  }
}
```

**SSE vs WebSocket**：
- SSE：单向（服务器→客户端），基于 HTTP，实现简单，适合AI流式输出
- WebSocket：双向，需要额外协议，适合聊天室、实时协作
